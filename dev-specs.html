Campaign
A single advertising flight. Groups a set of Creatives, a budget, dates, geography, and DSP/SSP distribution partners. The central planning object in the platform.
ConceptDescriptionRelates toTableExample
Campaign Belongs to a Client org and an Advertiser. Holds impression goals, budget range, geography, and flight dates. Linked to one or more DSP/SSP connections (partner_ids). Status drives the campaign lifecycle: draft → on-track → pacing → completed. A Moments Match analysis can be associated to a Campaign to inform which moments to buy.

Creative / Ad
A physical ad asset: can be a video, or a static image, and is the basis to build variants when applying templates (i.e. Sync, L-Bar, etc). Linked to a Campaign and an Advertiser. Template assignments (template_ids) declare which Creative Templates (ad formats) this creative will be applied with. The KERV AI uses the creative's content to run Moments Match and find contextually relevant inventory.
ConceptDescriptionRelates toTableExample
Creative A physical ad asset: YouTube embed, hosted video, or image. Linked to a Campaign and an Advertiser. Template assignments (template_ids) declare which Creative Templates (ad formats) this creative is compatible with. The KERV AI uses the creative's content to run Moments Match and find contextually relevant inventory.

Creative Template / Ad Type
An Ad Product, part of a stable library / offering, that enrich an asset with ads format and data, statically or dynamically (i.e. product catalogues, QR code, graphic elements, interactions).
ConceptDescriptionRelates toTableExample
Creative Template A global reference entry (not client-scoped) describing an ad format: e.g. Pause Ad, Sync, L-Bar, etc. Formats depend on the Media type (CTV, Web or Mobile). Defines creative specs (aspect, dimensions, interactivity) and technical aspects (tag, etc). Creatives can have one or more template. The Ad Format offering is always updating.

Moment
A contextual time-based inventory cluster matched with a piece of ad content, that is relevant to a specific ad. The core output of KERV's AI analysis engine — it groups detected signals / taxonomies, and can be assembled into Media Plans.
ConceptDescriptionRelates toTableExample
Moment A time-segment (or context cluster) in a content item where specific Taxonomy signals are active. The KERV AI scores each moment for contextual relevance to the analysed Creative. Moments carry estimated impressions (moment_est_impr), CPM, and dollar value. They are stored as JSONB within a Moments Match analysis, and selected moments form a Media Plan.

Taxonomy / Signals
The classification system used to describe what happens inside a piece of content. Signals are the language that bridges content analysis with ad targeting — every Moment is described by a set of Taxonomy signals.
ConceptDescriptionRelates toTableExample
Taxonomy / Signal A hierarchical label applied to content frames and moments: e.g. Sport > Football > Goal, Emotion > Excitement, Object > Car > Luxury. Signals are produced by the KERV AI analysis pipeline (tiers 0–3) and stored in the content analysis JSONB. Creative Templates declare which signal types they are designed to target, making Taxonomy the core matching vocabulary between inventory and ads.

Media Plan
An ordered selection of Moments with budget estimates, assembled within a Moments Match analysis. The deliverable output of the planning process — ready to be distributed to DSP/SSP partners.
ConceptDescriptionRelates toTableExample
Media Plan A named collection of selected Moments stored as JSONB inside a Moments Match analysis. Each plan entry carries: a list of moments (with impressions, CPM, dollar value), plus aggregate totals. One Moments Match analysis can hold multiple saved plans. A plan is linked to a Campaign when it is ready for distribution, and sent to connected DSP partners via the Distribute to DSP flow.

Partner
External technology platforms connected to KERV One for inventory supply and media buying. Two types: DSPs (demand-side) receive and execute media plans; SSPs (supply-side) expose inventory to DSPs.
TypeDescriptionRelates toTableExample

A DSP connection is created per Client org (optionally scoped to a specific Advertiser), and referenced on Campaigns via partner_ids. When a Media Plan is distributed, it is sent to the connected DSPs for activation.
